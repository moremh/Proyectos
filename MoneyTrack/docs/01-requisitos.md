# MoneyTrack - Documento de Requisitos

## 1. Información general

### Nombre del proyecto
MoneyTrack

### Descripción

MoneyTrack es una aplicación web de gestión financiera personal que permite
a los usuarios registrar, organizar y analizar sus ingresos y gastos.

La aplicación busca ayudar a las personas a tener un mejor control de su
dinero mediante estadísticas, reportes y herramientas de planificación.

---

# 2. Problema a resolver

Muchas personas tienen dificultades para controlar sus finanzas personales,
ya que no registran sus gastos, desconocen en qué utilizan su dinero y no
tienen una visión clara de su situación económica.

MoneyTrack busca solucionar este problema ofreciendo una herramienta simple,
visual y fácil de utilizar.

---

# 3. Objetivo general

Desarrollar una plataforma web que permita gestionar ingresos y gastos
personales, brindando información clara sobre los hábitos financieros del
usuario.

---

# 4. Objetivos específicos

- Registrar ingresos y gastos.
- Clasificar movimientos mediante categorías.
- Visualizar el balance financiero.
- Generar reportes y estadísticas.
- Permitir establecer objetivos de ahorro.
- Facilitar la toma de decisiones financieras.

---

# 5. Usuarios del sistema

## Usuario registrado

Puede:

- Crear una cuenta.
- Iniciar sesión.
- Registrar ingresos.
- Registrar gastos.
- Crear categorías.
- Consultar estadísticas.
- Gestionar objetivos de ahorro.

---

# 6. Alcance versión 1.0

La primera versión incluirá:

- Registro e inicio de sesión.
- Dashboard principal.
- Gestión de ingresos.
- Gestión de gastos.
- Categorías.
- Reportes básicos.
- Perfil de usuario.

---

# 7. Requisitos funcionales

RF-01:
El sistema debe permitir registrar usuarios.

RF-02:
El sistema debe permitir iniciar sesión.

RF-03:
El usuario debe poder registrar ingresos.

RF-04:
El usuario debe poder registrar gastos.

RF-05:
El usuario debe poder modificar y eliminar movimientos.

RF-06:
El sistema debe calcular automáticamente el balance.

RF-07:
El sistema debe mostrar estadísticas financieras.

---

# 8. Requisitos no funcionales

RNF-01:
La aplicación debe tener una interfaz intuitiva.

RNF-02:
Debe ser responsive para dispositivos móviles.

RNF-03:
La información del usuario debe mantenerse segura.

RNF-04:
El sistema debe estar preparado para futuras ampliaciones.

RNF-05:
El código debe estar organizado y documentado.
